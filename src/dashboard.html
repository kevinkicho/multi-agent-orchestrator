What is the "brain"?

Brain (brain.ts) is the LLM-driven orchestrator agent — it reasons about the project, reads worker responses, and issues commands (PROMPT, NOTIFY, NOTE, etc.) to steer the fleet.

The Orchestrator class (orchestrator.ts) is the infrastructure layer it commands — spawns OpenCode subprocesses, delivers prompts, collects messages. No LLM reasoning; just plumbing.

Same conceptual role, two layers: the brain thinks, the orchestrator dispatches.

What the brain does each round

  1. Reads every worker's latest messages and current status.
  2. Decides what to do next by calling its own LLM with a system prompt + conversation history.
  3. Issues commands: PROMPT a worker, PROMPT_ALL, NOTIFY, record a NOTE to persistent memory, or declare DONE.
  4. Waits for workers to finish, collects new responses, feeds them back into the next round.
  5. Saves memory: persists learnings and per-agent notes for future sessions.

Arbitrator role (truncation handling)

  • Brain's own response truncated: if the brain's LLM hits max_tokens mid-generation, any partial commands are not executed. A [VALIDATION] message asks the brain to retry more concisely — preventing half-formed PROMPTs from reaching workers.
  • Worker response truncated: an [ARBITRATOR NOTE] is appended to the worker's reply so the brain sees the response was cut off and can PROMPT the worker to continue.
  • Supervisor's own response truncated: same guard as the brain — skip commands, request a concise retry.
